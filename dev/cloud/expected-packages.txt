# Expected package set for the lean hosted cloud image (dev/cloud/mkosi.conf).
#
# dev/cloud/bootstrap.sh's lean check asserts the built image's package manifest
# matches this set EXACTLY: any package present in the image but not listed here
# (bloat — e.g. the recommends drift of #237), or listed here but absent from the
# image, fails `make build-cloud-image` and the CI cloud-image lane (#238). This
# is the leanness guard the old hardcoded cut list could not give — it catches
# ANY off-list package, not just a named handful.
#
# This is a lockfile: one package name per line, `#` starts a comment. Names
# only — versions are not compared, so a patch/minor bump does not churn this
# file; only a genuine change to the dependency graph does, and that change must
# be reviewed. ENVIRONMENT.md # How the profile is realized governs what the lean
# image carries; the appliance cuts (network-manager, avahi, samba, mergerfs,
# cryptsetup, tpm2-tools, openssh-server) must never appear below.
#
# nftables is in the set because docker-ce hard-Depends it (#241) — its proper
# fix/decision is tracked there; until then it is part of the lean set, not a
# leak.
#
# grub-common + grub-pc-bin (and their transitive libs — dmsetup, gettext-base,
# libbrotli1, libdevmapper1.02.1, libefiboot1t64, libefivar1t64, libfreetype6,
# libfuse3-4, libpng16-16t64) are in the set because the image is dual-firmware
# (#277): it boots UEFI via systemd-boot AND legacy BIOS via GRUB
# (BiosBootloader=grub), so it comes up on providers that present either firmware
# (Hetzner CX/Intel boots legacy BIOS; CPX/AMD boots UEFI). grub is the BIOS
# bootloader only — the UEFI path stays systemd-boot, so the systemd-boot* entries
# below remain.
#
# The concrete kernel-image package (linux-image-<abi>-amd64) is deliberately
# NOT listed: its name carries the ABI version, which changes on every kernel
# bump. The lean check normalizes it out of the diff on both sides; the stable
# meta-package linux-image-amd64 (below) is what guarantees a kernel is present.
#
# Regenerate when a package change is intended: run the lean check (the CI lane,
# or `make build-cloud-image` on a host where mkosi builds) and add the reported
# "UNEXPECTED" names / drop the reported "MISSING" names below.
#
